# Create PR

## Overview

Create a well-structured pull request with proper description, labels, and reviewers.

## Steps

1. **Prepare branch**
   - Ensure all changes are committed
   - Use the conventional commit format for commit messages.
   - Push branch to remote
   - Verify branch is up to date with base branch

2. **Create PR Title**
   - Craft a Clear Title: The title should be concise and accurately reflect the purpose of the PR, including the issue
     addressed and the type of changes made.

3. **Write PR description**
   - Summary of Changes: Briefly explain what was modified.
   - Context and Background: Provide necessary information for reviewers to understand the problem being solved and the
     rationale behind the chosen solution.
   - Why the Changes Were Made: Explain the reasoning and benefits of the modifications.
   - Testing Information: Include instructions on how to test the changes, including any necessary setup or
     dependencies.
   - Visual Aids: If applicable, include screenshots, GIFs, or videos to demonstrate UI changes.
   - Related Links: Link to any relevant issues, tickets, or discussions.

4. **Set up PR**
   - Create PR with descriptive title
   - Add appropriate labels
   - Assign reviewers
   - Link related issues
